# Knowledge Architecture - KM Analyst

**Work with the Developer to define:**

- The data schema for knowledge items (what fields does each knowledge entry have? tags,
  categories, author, date, type?).
- The taxonomy: how is knowledge classified in the app? Create a taxonomy diagram or table
  and commit it to /docs/km-architecture.md.
- The retrieval logic: how should users be able to search and filter knowledge? Write
  requirements (not code) for the Developer.
